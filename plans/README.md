# README для папки plans

Здесь живут **технические планы** реализации функций и фич.

## Как это работает

- Один план = одна функция или одна крупная задача
- Имя файла: `YYYY-MM-DD-название-функции.md`
- План разбит на фазы с статусами `[ ]` (не сделано) и `[x]` (сделано)
- После каждой сессии обновляй статус фаз

## Шаблон плана

```
# [Название функции]

Дата создания: YYYY-MM-DD

## Описание

[Что нужно сделать]

## Фазы реализации

### Фаза 1: [Название]
- [ ] Подзадача 1
- [ ] Подзадача 2
- [ ] Подзадача 3

### Фаза 2: [Название]
- [ ] Подзадача 1
- [ ] Подзадача 2

## Результат

[ ] Функция полностью реализована
[ ] Функция частично реализована
[ ] Функция не реализована

Что осталось: [Описать]
```
