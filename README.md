# README проекта

Добро пожаловать в Media Identity!

## Быстрый старт

1. Откройте [CLAUDE.md](CLAUDE.md) — это карта проекта
2. Прочитайте [.business/INDEX.md](.business/INDEX.md) — это оглавление бизнес-контекста
3. Откройте нужный раздел в `.business/` в зависимости от того, что вам нужно

## Структура

- **CLAUDE.md** — навигация по всему проекту
- **.business/** — весь бизнес-контекст (не в git)
  - **company/** — информация о компании
  - **audience/** — информация об аудитории
  - **products/** — информация о продукте
  - **marketing/** — маркетинг-стратегия
  - **goals/** — цели и KPI
  - **economics/** — финансовая информация
  - **assets/** — визуальные активы и бренд-гайдлайны
  - **история/** — рефлексия после каждой сессии
- **plans/** — технические планы реализации

## Основные правила

1. **Один план = одна функция** в папке `plans/`
2. **Всегда обновляй нужные файлы** в `.business/` когда меняется контекст
3. **После каждой сессии** добавляй рефлексию в `.business/история/`
4. **Язык общения:** русский

## Как начать работать

Откройте .business/INDEX.md и выберите нужный раздел.

## Как деплоить на Railway

Этот проект теперь содержит минимальный Node.js-сервер и конфигурацию для Railway.

1. Создайте репозиторий на GitHub.
2. В локальной папке выполните:
   - `git init`
   - `git add .`
   - `git commit -m "Deploy ready for Railway"`
   - `git branch -M main`
   - `git remote add origin <URL-репозитория-на-GitHub>`
   - `git push -u origin main`
3. Откройте https://railway.app и войдите в аккаунт.
4. Подключите GitHub-репозиторий в Railway.
5. Создайте новый проект на Railway и выберите деплой из GitHub.
6. Включите автодеплой для ветки `main`.

После этого Railway автоматически будет собирать и разворачивать сайт при каждом пуше.
